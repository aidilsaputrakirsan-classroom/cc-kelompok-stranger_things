# Retrospective — Milestone 1

## 🟢 Apa yang Berjalan Baik?

- Docker Compose berhasil digunakan untuk menjalankan beberapa service dalam satu perintah.
- Struktur project sudah mulai rapi karena backend, frontend, dan konfigurasi Docker dipisahkan.
- Dokumentasi awal seperti README 
- Anggota tim sudah mulai memahami alur kerja GitHub, branch, commit, pull request, dan review.
- Backend, frontend, dan database dapat dijalankan dalam container terpisah.

## 🔴 Apa yang Perlu Diperbaiki?

- Komunikasi tim perlu dibuat lebih teratur agar tidak ada pekerjaan yang tumpang tindih.
- Commit message perlu dibuat lebih konsisten menggunakan format seperti `feat:`, `docs:`, `fix:`, atau `chore:`.
- Testing fitur perlu dilakukan sebelum perubahan dikirim melalui Pull Request.
- Dokumentasi perlu terus diperbarui setiap ada perubahan pada project.

## 🔵 Action Items untuk Milestone 2

- Menggunakan branch terpisah untuk setiap fitur atau dokumentasi.
- Semua perubahan harus masuk melalui Pull Request sebelum digabung ke branch `main`.
- Minimal satu anggota tim melakukan review dan approve sebelum PR di-merge.
- Menambahkan checklist testing sebelum demo.
- Memastikan file `.env` tidak ikut ter-commit ke repository.
